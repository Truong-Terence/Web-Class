Le GRID

Grid en CSS :
On commencera par la création d'un grid-container avec le display : grid.

  • Colonnes grid-template-columns en px. Pour 3 colonnes, décrire 3 mesures
  • Rangées grid-template-rows (px)
  • Rangées auto grid-auto-rows(px) pour définir la valeur par défaut
  • Gap grid-column-gap ou grid-row-gap ou encore grid-gap (px)
  • fr : l'unité fr s'adapte à l'espace disponible sur l'écran.
  • repeat : grid-template-columns : 100 px repeat(2, 1fr 2fr)
  • minmax grid-template-columns : minmax(200px, 250px) 1fr 1fr
    on peut utiliser minmax dans le grid auto : minmax(60px, auto)
  • auto-flow place les éléments du grid par colonne ou par rangée. grid-auto-flow : column
  • order fonctionne comme sur flexbox. order : x
  • auto fill grid-template-columns: repeat(auto-fill, minmax(50px 1fr))
  • auto fit grid-template-columns: repeat(auto-fit, minmax(50px 1fr)) remplit le container
  • auto-placement grid-column-end ou grid-row-end : span x
  • gap filling Dans le grid : grid-auto-flow : dense;
  • placement grid-row : x
    1 est la première ligne, -1 est la dernière donc grid-column : 1 / -1
  • Line-based positionning .class:nth-child(x) {
    grid-column-start : x;
    grid-column-end : -x;
    }
    ou grid-column : x / -x;
    ou grid-column : x / span x;
  • Named linesgrid-template-columns : 2fr [content-start] 4fr [content end];
    grid-template-rows : auto [content-start] auto [content-end]. On peut maintenant utiliser ces noms pour placer les éléments du grid sur l'enmplacement nommé.
  • justify justify-items / align-items : stretch, start, end, center;
  • align-self aligne-self pour les éléments dans le grid area.
  • L'alignement du grid On peut ajouter une border au grid, et utiliser justify-content ou align-content avec les paramètres de flexbox
  • areas grid-template-areas permet de placer les éléments nommés avec grid-area. Un point "." ou "..." place un élément vide.
  • Les noms définits dans grid-area peut s'utiliser pour placer d'autres éléments autour ou même sur un élément existant.
  • display-content change des enfants non-directs en enfants directs.
  • stacking est possible, on peut aussi utiliser z-index.
  • Anonymous items si c'est un string, agit comme un grid item.
  • Feature Queries : @supports(diplay : grid) {...} permet d'affichier ou non un item selon le browser.
  • Track Sizing width : min-content; permet d'ajuster la largeur sur le mot le plus long de la string. max-content s'ajuste à la taille du string(risque d'overflow).
    fit-content(em or %) stretch autant qu'il peut jusqu'au max em ou %.
