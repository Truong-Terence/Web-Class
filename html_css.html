Le langage du web

Le HTML

HTML Hyper Text Markup Language
Langage standard du web. Il fonctionne sur un système de "balises" lui permettant une certaine mise en forme et l'insertion d'objets (images, vidéos, ...).
Balise
Les balises indiquent au navigateur comment afficher un objet, un élément en l'encadrant ainsi : <balise>élément</balise> pour les balises fermantes.
Les balises orphelines quant-à elle appliquent leur effet lorsqu'on les déclare. Les navigateurs n'affichent pas les balises.

I - Mettre en valeur

Que ce soit un texte, une phrase ou un mot, on peut lui approrter une différence de style ou de taille. On les classe dans des catégories qui ont des paramètres prédéfinis mais modifiables.

  • les titres sont classifiables de h1 à h6.
  • les valeurs :
    • em (emphasized) pour donner un peu de valeur.
    • strong pour donner une forte valeur.
    • mark pour surligner et faire ressortir visuellement.

II - Faire des listes

Pour structurer des éléments d'informations, on a le choix entre 2 balises :

  • ul (unordored list).
  • ol (ordored list).
Le listing s'établie en 3 étapes :
  1. L'ouverture avec la balise de son choix.
  2. L'énumération des éléments entre les balises <li> et </li>.
  3. La fermeture de la balise de liste.
Un 3ème type de liste existe, on le connait sous le terme "définition".
On utilise dl pour déterminer la liste puis dt pour déterminer le terme et enfin dd pour la/les définition/s.

III - Créer des liens

Les balises peuvent contenir des attributs dans la balise ouvrante qui permettent une certaine souplesse des balises. Pour créer un lien on utilise la balise a à laquelle on ajoute l'attribut href pour indiquer l'endroit d'atterissage du lien, ainsi : <a href ="point_de_chute">nom_lien</a>.
Deux types de liens s'offrent à nous :

  • Les liens ablolus sont des liens existants commet une page html.
  • Les liens relatifs sont des liens ayants pour point de départ le fichier dans lequel on se trouve.
Si le fichier à joindre est dans le même dossier, pas de soucis on indique son nom dans href. Pour aller dans un sous-dossier, il faut alors énumérer les dossiers en les séparant par un /. Et enfin pour remonter dans un dossier parent on tape "..".
Pour créer une ancre on utilisera l'attribut id dans la balise ouvrante. Cet id que l'on réutilisera dans href à l'aide d'un #ancre.
Un lien vers une ancre d'une autre page ? Il suffit d'additionner les deux méthodes en tapant "point_de_chute#ancre" dans href.

IV - Images, Audio et vidéo :

Les Images:

Les formats:
  • JPEG: pour les photos.
  • PNG: pour toutes les autres illustrations.
  • GIF: similaire au PNG, plus limité en nombre de couleurs mais qui peut être animé.
Insérer une image:

Pour insérer une image il faut utiliser la balise orpheline <img src="" alt="">

Important : Une image insérée dans du HTML doit toujours etre comprise dans une balise P.
Important : alt : cela signifie « texte alternatif ». On doit toujours indiquer un texte alternatif à l'image, c'est-à-dire un court texte qui décrit ce que contient l'image. Ce texte sera affiché à la place de l'image si celle-ci ne peut pas être téléchargée, ou dans les navigateurs de personnes handicapées (non-voyants) qui ne peuvent malheureusement pas « voir » l'image. Cela aide aussi les robots des moteurs de recherche pour les recherches d'images. Pour afficher une info-bulle il faut rajouter l'attribut title="".

Les figures:

Si une image illustre le texte (et n'est pas seulement décorative), il est conseillé de la placer au sein d'une balise <figure>. La balise <figcaption> permet d'écrire la légende de l'image.

L'Audio:

La balise <audio src=""></audio> permet uniquement de télécharger le contenu audio il faut compléter la balise par les attributs ci-dessous :

  • Controls: pour ajouter les boutons play/pause et la barre de défilement.
  • Witdh: podifier la largueur de la barre.
  • Loop: boucle audio.
  • Autoplay: lecture dès le lancement de la page.
  • Preload: indique si la musique peut être préchargée dès le chargement de la page ou non. Cet attribut peut prendre les valeurs : auto(par défaut) : le navigateur décide s'il doit précharger toute la musique, uniquement les métadonnées ou rien du tout, metadata: charge uniquement les métadonnées (durée, etc.), none: pas de préchargement. Utile si vous ne voulez pas gaspiller de bande passante sur votre site.

Si jamais le navigateur ne reconnais pas le format du fichier il faut utiliser la balise comme ci-dessous :
<audio controls> <source src=""> </audio>.

Les Vidéos:

Pour insérer une video, il suffit d'utiliser la balise <video src=""></video>.
Comme pour l'audio, il faut lui rajouter les mêmes attributs + un attribut supplémentaire :

  • Poster: image à afficher à la place de la vidéo avant son lancement.

La structuration:

En général, une page web est constituée d'un en-tête (tout en haut), de menus de navigation (en haut ou sur les côtés), de différentes sections au centre… et d'un pied de page (tout en bas).

  • Header, on y trouve le plus souvent un logo, une bannière, le slogan de votre site
  • Nav doit regrouper tous les principaux liens de navigation du site.
  • Section sert à regrouper des contenus en fonction de leur thématique. Elle englobe généralement une portion du contenu au centre de la page.
  • Article sert à englober une portion généralement autonome de la page. C'est une partie de la page qui pourrait ainsi être reprise sur un autre site.
  • Aside, conçue pour contenir des informations complémentaires au document que l'on visualise. Ces informations sont généralement placées sur le côté (bien que ce ne soit pas une obligation).
  • Footer se trouve en général tout en bas du document. On y trouve des informations comme des liens de contact, le nom de l'auteur, les mentions légales, etc.

V - Les types de balises:

Les balises paires:

Elles s'ouvrent, contiennent du texte, et se ferment plus loin. Voici à quoi elles ressemblent : <titre>votretexte</titre> On distingue une balise ouvrante <titre> et une balise fermante </titre> qui indique que le titre se termine. Cela signifie pour l'ordinateur que tout ce qui n'est pas entre ces deux balises… n'est pas un titre.
Ceci n'est pas un titre
<titre>Ceci est un titre</titre>.

Les balises orphelines:

Ce sont des balises qui servent le plus souvent à insérer un élément à un endroit précis (par exemple une image). Il n'est pas nécessaire de délimiter le début et la fin de l'image, on veut juste dire à l'ordinateur « Insère une image ici ». Une balise orpheline s'écrit comme ceci : <image />

Les attributs:

Les attributs sont un peu les options des balises. Ils viennent les compléter pour donner des informations supplémentaires. L'attribut se place après le nom de la balise ouvrante et a le plus souvent une valeur, comme ceci :
<balise attribut="valeur">.

VI - Les formulaires:

Un formulaire est une zone interactive de la page, dans laquelle les visiteurs peuvent saisir des informations. On délimite un formulaire avec la balise <form></form> , à laquelle il faut ajouter deux attributs : method (mode d'envoi des données) et action (page vers laquelle le visiteur sera redirigé après envoi du formulaire, et qui traitera les informations).

Les éléments des formulaires:

Les éléments des formulaires doivent être insérés dans la balise <input>.
La valeur de son attribut type permet d'indiquer quel type de champ doit être inséré :

  • Text: zone de texte.
  • Password: zone de texte pour mot de passe.
  • Hidden: champ caché.
  • Tel: numéro de téléphone.
  • Checkbox: case à cocher.
  • Radio:bouton radio (un seul choix).
  • Button: bouton.
  • Reset: remise à zero du formulaire.
  • Image: image cliquable.
  • File: fichier.
  • Email: adresse mail.
  • Url: adresse URL/URI.
  • Search: champ de recherche.
  • Date: date.
  • Time: heure.
  • Datetime-local: date et heure.
  • Month: mois.
  • Week: semaine.
  • Number: valeur numérique.
  • Range: valeur numérique d'un intervalle.
  • Color: une couleur RVB.

Label:

La balise <label> permet d'écrire un libellé. On l'associe à un champ de formulaire avec l'attribut for="", qui doit avoir la même valeur que l' id du champ de formulaire.
<label for="pseudo">Votre pseudo :</label>
<input type="text" name="pseudo" id="pseudo"/>

Les attributs:

Pour faciliter la vie de l'utilisateur nous pouvons : rendre un champ obligatoire avec l'attribut required, faire en sorte qu'il soit sélectionné par défaut avec autofocus, donner une indication dans le champ avec placeholder

Le CSS

CSS Cascading Style Sheets :
Langage de stylisation d'un document HTML. Il s'agit d'une liste de règles comportant un ou des selecteurs et un block ou sont déclarées les règles.
Sélécteur
Les sélecteurs déclarent à quelle partie du balisage un style s'applique.

I - Les intégrations du CSS

Le CSS peut être écrit à trois endroits différents :

  • Dans un fichier .css (méthode la plus recommandée).
  • Dans l'en-tête <head> du fichier HTML.
  • Directement dans les balises du fichier HTML via un attribut style (méthode la moins recommandée).

Dans un fichier .CSS:

Il faut d'abord crée son fichier .css et le relier au fichier .hmtl grâce à la balise que l'on doit placer dans le <head></head>
<link type="text/css" rel="stylesheet" href="styles/css/style.css">

Dans l'en-tête du HTML:

Il suffit d'inscrire les règles CSS dans une balise <style> placée dans le <head>.
Cette méthode n'est pas conseillée.

Directement dans les balises (fortement non recommandé):

Ajouter un attribut style à n'importe quelle balise. Vous insérerez votre code CSS directement dans cet attribut <p style="color: blue;">Bonjour et bienvenue sur mon site !</p>


En CSS, le style d'une balise sera transmit aux balises à l'intérieur de celle-ci. C'est pour ça que le style de la balise <body> est transmit aux balises <p> et <h1>. Toute la page héritera alors des styles, il faudra créer des exceptions pour palier à ça.

II - la sélection des balises:

Les sélecteurs de balise:

Ce sélecteur simple permet de cibler les éléments qui correspondent au nom indiqué.

  • Syntaxe: nom_element.
  • Exemple: input permettra de cibler n'importe quel élément <input>.

Les sélecteurs de classe:

Ce sélecteur simple permet de cibler les éléments en fonction de la valeur de leur attribut class.

  • Syntaxe: .nomclasse.
  • Exemple: .index permettra de cibler n'importe quel élément qui possède la classe index (définie avec un attribut class="index").

Les sélecteurs d'identifiant:

Ce sélecteur simple permet de cibler un élément d'un document en fonction de la valeur de son attribut id. Dans un document, il ne doit y avoir qu'un seul élément pour un identifiant donné.

  • Syntaxe: #valeurid.
  • Exemple: #toc permettra de cibler l'élément qui possède l'identifiant toc (vraisemblablement défini avec un attribut id="toc").

Le sélecteur universel:

Ce sélecteur permet de cibler tous les nœuds d'un document. Il existe également une variante pour ne cibler qu'un seul espace de noms et une variante pour cibler tous les espaces de noms.

  • Syntaxe: * ns|* *|*.
  • Exemple: * permettra de cibler tous les éléments du document.

Les sélecteurs d'attribut:

Ce sélecteur simple permet de cibler des éléments d'un document en fonction de la valeur d'un de leurs attributs.

  • Syntaxe: [attr] [attr=valeur] [attr~=valeur] [attr|=valeur] [attr^=valeur] [attr$=valeur] [attr*=valeur].
  • Exemple: [autoplay] permettra de cibler tous les éléments qui possède l'attribut autoplay défini (quelle que soit sa valeur).

III - Les Class et ID:

Comment faire pour que certains paragraphes seulement soient écrits d'une manière différente ? On pourrait placer le code CSS dans un attribut style sur la balise que l'on vise mais ce n'est pas recommandé (il vaut mieux utiliser un fichier CSS externe). Pour résoudre le problème, on peut utiliser ces attributs spéciaux qui fonctionnent sur toutes les balises :

  • Les Class Class=""
  • les ID Id=""

Class:

Class permet de cibler plusieurs balises du code html.
Il suffit de l'intérgrer au code HTML grâce à l'attribut class=".nom_classe"
Par la suite il faudra cibler l'attribut class dans le fichier html en utilisant le sélécteur de classe .nom_classe

ID:

ID reprend le même principe que class, SAUF qu'il est utilisable qu'une seule fois dans le code.
id="nom_id"
Pour cibler une ID dans le CSS il faut utiliser le selecteur #nom_id

IV - Formatage du texte :

La taille :

Pour modifier la taille du texte, on utilise la propriété CSS font-size. Il y a plusieurs techniques:

- Indiquer une taille absolue : en pixels, en centimètres ou millimètres. Cette méthode est très précise mais, il est conseillé de ne l'utiliser que si c'est absolument nécessaire, on risque d'indiquer une taille trop petite pour certains lecteurs.
Pour indiquer une taille absolue, on utilise généralement les pixels. Pour avoir un texte de 16 pixels de hauteur, il faut écrire : font-size: 16px;

- Indiquer une taille relative : en pourcentage, en « em » ou en « ex », cette technique a l'avantage d'être plus souple. Elle s'adapte plus facilement aux préférences de taille des visiteurs.
Il y a plusieurs moyens d'indiquer une valeur relative. Par exemple écrire la taille avec des mots en anglais comme ceux-ci :

  • xx-small: minuscule.
  • x-small: très petit.
  • small: petit.
  • medium: moyen.
  • large: grand.
  • x-large: très grand.
  • xx-large: gigantesque.
- Définir la taille en "em".
  • 1em: le texte a une taille normale.
  • 1.3em: pour grossir le texte, inscrire une valeur supérieure à 1.
  • 0.8em: pour réduire le texte, inscrire une valeur inférieure à 1.
- Définir la taille en pourcentage font-size: 8%.
- Définir la taille en "ex", plus petit que em.

La police:

La Police utilisée par defaut par les navigateur est la sans-serif.
Pour ajouter une police, il faut suivre la procédure ci-dessous:
@font-face {
font-family: 'Police-name';
src: url('police-name.eot');
}
On pourra par la suite définir la police d'un élement, par exemple d'une section:
section {
font-family : 'police-name.eot'
}
- Les différents formats de polices:

  • .TTF (TrueType Font): fonctionne sur IE9 et tous les autres navigateurs.
  • .EOT (Embedded OpenType): fonctionne sur Internet Explorer uniquement, toutes versions.
  • .OTF (OpenType Font): ne fonctionne pas sur Internet Explorer.
  • .SVG (SVG Font): le seul format reconnu sur les iPhone et iPad pour le moment.
  • .WOFF (Web Open Font Format): nouveau format conçu pour le Web, qui fonctionne sur IE9 et tous les autres navigateurs.

Décorations de texte:

- font-style:

  • italic: italique.
  • oblique: le texte sera passé en oblique (les lettres sont penchées, le résultat est légèrement différent de l'italique proprement dit).
  • normal: texte normal, permet d'annuler un texte en em.

- font-weight:

  • bold: texte en gras.
  • normal: par défaut.

- text-decoration:

  • underline: souligné.
  • line-through: barré.
  • overline: ligne au-dessus.
  • blink: clignotant. Ne fonctionne pas sur tous les navigateurs (Internet Explorer et Google Chrome, notamment).
  • none: aucune décoration (utile pour retirer l'underline d'un lien.

V - Les alignements:

À l'instar des logiciels de traitement de texte, la propriété text-align comprend les 4 arguments "left", "right", "center", "justify".
Le CSS permet également de faire "flotter" un élément autour d'un texte, c'est un habillage.
Dans le fichier HTML on place l'élément flottant avant le texte puis on utilise la propriété CSS float en indiquant l'argument right ou left.
On peut retirer,clear le flottant avec les arguments left, right ou both.

VI - Les couleurs !

Comment ça fonctionne ?

  • La teinte est le plus basique des termes de couleur et désigne la couleur d’un objet. Lorsque nous disons «bleu», «vert» ou «rouge», nous parlons de teinte.
  • La chrominance, une teinte avec une chrominance élevée n'a pas de noir, de blanc ou de gris ajouté. On se réfère à la couleur pure.
  • La saturation fait référence à la façon dont une teinte apparaît dans des conditions d'éclairage particulières (teintes faibles vs fortes ou pâles vs pures).
  • La valeur se réfère à la luminausité. Plus la valeur est haute, plus la couleur est claire.
  • Les tons sont créés lorsque du gris est ajouté à une teinte, ça peut ajouter de la douceur et de l'élégance.
  • Les nuances sont crées en ajoutant du noir à une teinte. Il est préférable de combiner des nuances avec des teintes ou des neutres plus clairs pour éviter un look trop sombre et trop lourd.
  • Une teinte se forme lorsque du blanc est ajouté à une teinte, l'éclaircissant. Les teintes très claires sont parfois appelées pastels.

On regroupe les couleurs par type, chacune d'elle peut avoir une signification même si elle peut varier selon les cultures.

Les couleurs chaudes :

Rouge - (couleur primaire)
Tant associée au feu, à la violence, au danger, à la guerre que l'amour et la passion.
Dans l'Ouest du Monde elle a un rapport à la chance, la prosperité, le bonheur ou la chance.
Le reste du Monde peut lui attribuer le deuil ou le marriage. Elle est aussi la couleur de (RED) campaign et du communisme.
Orange - (couleur secondaire)
Orange est une couleur liée au dynamisme, la créativité et à l'automne.
Grâce au fruit du même nom, on peut l'associer à la santé. C'est une couleur friendly.
Jaune - (couleur primaire)
Considérée comme la plus lumineuse et énergisante des couleurs grâce à son rapport au Soleil, à l'espoir et au bonheur. Pas aussi puissant que le rouge, on utilise aussi le jaune pour annoncer un danger.
On peut associer le jaune à la lâcheté et la tromperie. Dans certains pays d'Asie il représente le courage, la couleur marchande ou encore le deuil.

Les couleurs froides :

Vert - (couleur secondaire)
Vert est un retour à la Terre, un renouveau, le calme mais aussi la jalousie ou le manque d'expérience.
Le vert sombre est plus stable que le vert clair, qui lui est energiseant - surtout associé au jaune.
Bleu - (couleur primaire)
Aussi triste que calme et responsable, le bleu clair peut être frais et amical tandis que le bleu foncé fort et stable.
Violet - (couleur secondaire)
Auparavant, le violet était cher à fabriquer et donc étroitement lié à la royauté et la noblesse. On l'associe aujourd'hui à la créativité de par son origine : le mélange de bleu et de rouge hérite de certains attributs.

Les couleurs neutres :

Noir
Le noir est très puissant, utilisé positivement on tend vers l'élégance, le pouvoir. Le côté négatif du noir est son association à la mort et au mystère.
Blanc
Opposé au noir, le blanc représente la pureté, la propreté et la vertu. C'est une couleur qui se marrie avec n'importe quelle autre.
Gris
Le gris peut être considéré comme déprimant. Le gris clair peut remplacer le blanc tandis que le gris foncé peut remplacer le noir. C'est une couleur sophistiquée qui peut servir dans le milieu du design.
Marron
Le marron est associé à la Terre, au bois et à la pierre. C’est une couleur neutre et chaleureuse. Le brun peut être associé à la fiabilité, à la fermeté et au caractère terreux. Cela peut également être considéré comme terne.
Le marron est souvent utilisé pour le fond, la texture qui rappelera le bois.
Beige
Le beige est capable de prendre un ton chaud ou froid selon son environnement. Il est souvent réservé aux couleurs de fond.
Il peut symboliser la piété.
Crème et Ivoir
Ils sont généralement calmes et peuvent souvent évoquer un sens de l’histoire tout en étant légèrement plus chaleureux que le blanc. L'ivoir peut être utilisé pour éclaircir sans imposer le contraste du blanc.

Il y a deux méthodes pour la notation des couleurs :
  • Hexadécimale, soit un nom de couleur en hexadécimal, cela ressemble à : #FF5A28. Pour faire simple, c'est une combinaison de lettres et de chiffres qui indiquent une couleur. On doit toujours commencer par écrire un dièse (#), suivi de six lettres ou chiffres allant de 0 à 9 et de A à F. Ces lettres ou chiffres fonctionnent deux par deux. VII - Le background

Changer la couleur d'arrière-plan, mettre une image d'arrière-plan, ... Utilisons la super propriété background qui regroupe 8 propriété de backround:

  • background-color spécifie la couleur d'arrière-plan.
  • background-image spécifie une ou des images d'arrière-plan. On peut aussi mettre plusieurs images de fond en séparant les déclarations par des virgules
  • background-position spécifie la position de background-image.
  • background-size spécifie la taille de background-image.
  • background-repeat spécifie si on souhaite répéter et combien de fois l'image d'arrière-plan.
  • background-origin spécifie la zone de positionnement des images d'arrière-plan.
  • background-clip spécifie la zone de peinture des images d'arrière-plan.
  • background-attachment spécifie si les images d'arrière-plan sont fixes ou défilent avec le reste de la page.
Il faut juste espacer les attributs et il n'est pas obbligatoire de mettre toutes les propriété.
On peut y ajouter la propriété opacity qui varie de 0 à 1 pour la transparence, ce qui nous amène vers l'amélioreation de la notation RGB pour RGBa, le "canal alpha" correspondant à la transparence.

VIII - Les bordures

Il existe de nombreuses bordures en CSS et donc de nombreuses propriétés border-x mais on peut utiliser la super-propriété border avec jusqu'à 3 valeurs :

  • La largeur (en px, em, %, ...).
  • La couleur.
  • Le type de bordure (solid, dotted, dashed, double, groove, ridge, inset, outset) Elle est de base sur none.
Si on souhaite être plus précis on peut rajouter à border un -top, -bottom, -left, -right.
La propriété border-radius permet d'arrondir les coins, on peut préciser 1 ou 4 valeurs (en px) pour appliquer l'effet dans cet ordre : haut-gauche, haut-droite, bas-droite, bas-gauche.

IX - Les ombres

  • box-shadow pour les ombres sur un bloc entier.
  • text-shadow pour les ombres de texte.
Ces propriétés fonctionnent de la même manière et prennent 4 valeurs dans l'ordre qui suit :
  1. Décalage horizontale de l'ombre.
  2. Décalage vertical de l'ombre.
  3. Adoucicement du dégradé.
  4. Couleur de l'ombre.
  5. inset pour que l'ombre soit à l'intérieur du bloc - facultatif.

X - Les dimmensions

Les dimmensions sont modifiables avec width et height (px, em ou %). On peut demander des valeurs minimales et maximales avec min_width, max_width, min_height, max_height.
Rq : de base un bloc utilise 100% de la largeur disponible.
Il y a deux type de marge:

  • margin pour les marges extérieur
  • padding pour les marges intérieurs
Par défault il n'y a pas de marge intérieur mais une extérieur, mais celà dépend de la balise. Le plus simple est d'utiliser des propriétés précises: top, bottom, left, right sur margin ou padding avec 4 valeurs de pixels dans l'ordre suivant : haut, droite, bas, gauche.
Lorsqu'on ne connaît pas la résolution du visiteur il est pratique d'avoir un design centré, on donne alors une largeur au bloc avec width et on indique que l'on veut une margin auto.
L'overflow arrive à la rescousse lorsque les mesures sont trop précises et éviter un dépassement :
  • visible, par défault il sort du bloc.
  • hidden coupera purement et simplement.
  • scroll, il sera coupé mais une barre de défilement nous aidera à lire le bloc.
  • auto dépendra du navigateur, en général il utilisera scroll.
Si un mot ou un URL est trop long pour la largeur, on peut utiliser word-wrap : break-word pour le couper et forcer le retour à la ligne.

XI - Felxbox

Avant le module Flexbox il y avait 4 modes de diposition :

  • block pour les sections.
  • inline pour le texte.
  • table pour les tables à 2 dimmensions.
  • positioned pour la position explicit d'un élément.
Flexbox agit comme si on positionnait des boîtes dans une plus grosse boîte. Pour commencer, il faut donc définir un container flex.
  • Les containers, deviennent flex en utilisant display: flex;. Il peut alors avoir différentes propriétés :
    • flex-direction définit donc la direction : column, column-reverse, row et row-reverse.
    • flex-wrap définit si les flex-items iront à la ligne ou non : wrap, nowrap et wrap-reverse.
    • flex-flow regroupe flex-direction et flex-wrap.
    • justify-content est utilisée pour pour aligner des flex-items : center, flex-start, flex-end, space-between ou space-around.
    • align-items est utilisée pour aligner des flex-items sans modifier les flex-items : center, flex-start, flex-end, stretch(défaut), baseline.
    • align-content est utilisé pour aligner les flex-lines : space-between, space-around, stretch (défaut), center, flex-start, flex-end.
    Pour un centrage parfait, il suffit de fixer justify-content et align-items sur "center".
  • Les flex itemssont les enfants directs d'un flex container, ils ont les propriétés suivantes :
    • order définit l'ordre des flex-items.
    • flex-grow définit la croissance d'un élément flexible par rapport au reste des éléments flex. La valeur de base est 0.
    • flex-shrink définit dans quelle mesure un élément flexible se rétrécira par rapport au reste des éléments flex. La valeur par défaut est 1.
    • flex-basis définit la longueur initiale d'un flex-item (px).
    • flex comprend flex-grow, flex-shrink et flex-basis.
    • align-self définit l'alignement de l'élément sélectionné à l'intérieur du conteneur flexible. La propriété align-self remplace l'alignement par défaut définit par la propriété align-items du conteneur.
Il ne faut pas oublier l'importance d'un site web "flexé" car c'est un moyen efficace de faciliter le MédiasQueries.
